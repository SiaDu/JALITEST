## Eye performance tags

ExpreGaze treats eye performance as part of the same actor-style readable annotation as gaze / mask / heart.

LLM outputs one unified annotation:

```text
g   = gaze
m   = JALI mask / surface expression
h   = JALI heart / internal undercurrent
l   = lid_state
pb  = performative blink
bs  = blink_suppression
```

LLM should output `l / pb / bs` only when they are acting choices. It must not output regulatory blinks.

### Actor-first prompt policy

Before inserting tags, the LLM must build a concise character performance profile:

```text
character_read:
- apparent age / performance age
- social role and power position
- temperament
- emotional expressiveness
- acting style
- baseline energy
- baseline blink tempo
- baseline lid_state

performance_strategy:
- how the character uses gaze
- how the character uses eyelids
- how the character uses blink or blink suppression
- where the performance strategy actually changes
```

This prevents `lid_state` from becoming a second emotion label and prevents performative blink from becoming procedural eye noise.

### lid_state scale

```text
-9: fully wide, manic / possessed / extreme fear
-7: feverish, unstable, obsessive
-5: intense, neurotic, over-alert
-3: shocked / angry / excited
-2: heightened attention
-1: alert but controlled
 0: neutral
 1: relaxed
 2: soft / warm / receptive
 3: guarded / impatient / skeptical
 4: sleepy / withdrawn / flat dislike / emotionally closed
```

`lid_state` is a state-change tag:

```text
<l01=-1>
<l02=2>
```

Use `lid_state` only for sustained eyelid baseline / acting strategy changes. For a short clip, prefer one baseline state plus 2-3 major changes.

Dismissive blink belongs to lid_state, not performative blink:
- use `lid_state=3` or `lid_state=4`.

Anti-blink also belongs to lid_state + blink_suppression:
- use `lid_state=-3` or `lid_state=-5`
- add `<bs##=SUPPRESS>` when not blinking is the performance choice.

### performative blink modes

Only three categories are allowed in v1:

```text
1. frequent small blinks / double blink / blink cluster
   心虚 / 紧张 / 不确定

2. eye close / eye close hold
   思考、逃避、不愿直视、压住情绪、祈祷/神秘感、疲惫

3. slow blink
   认知检索、回忆、编故事、寻找措辞
```

Readable tags:

```text
<pb01=DOUBLE_BLINK-NERVOUS>
<pb02=BLINK_CLUSTER-UNCERTAIN>
<pb03=EYE_CLOSE_HOLD-AVOIDANCE>
<pb04=SLOW_BLINK-COGNITIVE_RETRIEVAL>
```

`pb` is an anchor event, not a state-change event. It does not persist until the next `pb`.

Use performative blink sparingly. A controlled / authoritative / older theatrical character should usually blink less, not more.

### blink_suppression

```text
<bs01=SUPPRESS>
<bs02=ALLOW>
```

`SUPPRESS` disables generated regulatory blink until `ALLOW` or clip end.

Use it for:
- shock
- anger
- manic / heightened attention
- hypnotic stare
- threat / control
- intense mystical conviction

### regulatory blink

Regulatory blink is not generated by LLM. It is generated by code after TextGrid timing resolve.

Rules kept conceptually:

```text
1. When a gaze event happens:
   add a blink 0-2 frames before the gaze transition or during the transition.

2. When mask / heart / expression / lid_state changes:
   add a blink around the state change as a facial reset.

3. If there is no blink for more than 4-6 seconds:
   add a very subtle blink.

4. Respect cooldown.

5. Respect blink_suppression.
```

Current v1 default is conservative:

```text
- gaze-change regulatory blinks: disabled by default
- mask/heart/lid-state reset blinks: disabled by default
- long no-blink subtle blinks: enabled by default
```

Reason: until the actor prompt is stable, transition blinks can fight performative blinks and make the eyes noisy. Reset blinks can be re-enabled later with CLI flags.

### Outputs

The compiler/resolver still starts from one unified readable annotation, but exporters split execution files:

```text
annotated_for_jali.txt                  # only JALI-compatible mask / heart
gaze_events_resolved.json               # Maya gaze adapter
eye_performance_events_resolved.json    # Maya blink/lid adapter
debug_full_annotation.txt               # full readable annotation
performance_events_resolved.json        # debug all-in-one
```

### Maya eye adapter

JALI blinks are removed and replaced by ExpreGaze blink.

The current eyelid control is:

```text
LIDS_jSync_plusMinus.Down_upLids_jSync
```

Before applying new blink/lid animation:
- clear old keys on this attribute
- disable JALI automatic blinks where possible:
  - `jSync1.calculate_blinks = false`
  - `jSync1.blink_frequency = 0`
  - `jSync1.long_blink_variation = false`
